Accessible Web Design

Content on the web should be universally accessible. To help developers, there are principles outlined in the Web Content Accessibility Guidelines (WCAG) that should be adhered to.

What are POUR Principles?

Perceivable

Everything on a webpage should be perceivable to the user, including the UI (User Interface) and page content. To acheive this, content must be perceivable through multiple means. This way, users who have one or more senses restricted are still aware of the content available.

    Example Problems
  • Audio content is not perceivable to deaf users
  • Images are not perceivable to blind users
  • Information conveyed through color is not perceivable to colorblind users
    Potential Solutions
  • Audio content should have a text alternative, such as transcrips and captioning on videos
  • Images should have alt text
  • Information should never rely soley on color to distinguish it

Operable

Users should be given multiple means to interact with a webpage. The UI should be accessible to everyone, whether they use a mouse, keyboard, mouth stick, or other tool to navigate a website. Under no circumstances should the UI require an action that a user is not able to perform.

    Example Problems
  • A button requires a click to perform an action
  • Content is timed, creating an obstacle for some users
  • Animation and flashing content prevents some users from being able to use the webpage
    Potential Solutions
  • Operable modalities should include clicks, keyboard entries, pointer gestures, etc
  • Preferable content would not be timed, but if so should allow for pauses, suppressed interruptions, and timeout warnings
  • Nothing should flash more than 3 times per second, and animation can be disabled

Understandable

Website information, functions, and navigation should be intuitive for the user. Everything they need to know about the page, including content and operation, should be available from the page.

    Example Problems
  • The website uses tehnical jargon, slang, or abbreviations which are uninterpretable to the user
  • There is no perceivable change in content when focused on
  • Forms that require input do not specify what is needed from the user or offer validation/correction
    Potential Solutions
  • Websites should use language appropriate targetted language so that it can be undestand by the website's audience
  • Users should be able to understand where they are on the page, and content focused on should indicate location
  • Forms should identify errors and suggest solutions with clear lables and instructions for users

Robust

A website must be able to fit the current needs of the user, as well as anticipate and adapt to future needs. It should also be able to be reliable interpreted by the user through various means, including assistive technologies. Content should remain accessible and interpretable even as technology evolves.

    Example Problems
  • Website not compatible with older browsers
  • Website design not standard across platforms
  • Website not supported by multple assistive technologies
    Potential Solutions
  • HTML should be compatible across browser and OS versions
  • Website should be scalable and compatible with all platforms
  • Website should account for various user agents and be supported by all
